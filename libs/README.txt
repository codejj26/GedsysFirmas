Wacom STU SDK DLLs - Installation Instructions
===============================================

This directory should contain the Wacom STU SDK native DLLs required for
real signature capture functionality with Wacom STU-430 tablets.

REQUIRED FILES
--------------
Place the following files in this directory:

1. wgssSTU.dll - Main Wacom STU SDK DLL (64-bit version)
2. wgssSTU_x64.dll - Alternative 64-bit version (if available)

HOW TO OBTAIN THE SDK
----------------------
The Wacom STU SDK can be downloaded from:

Official Documentation:
https://developer-docs.wacom.com/docs/stu-sdk/

Direct Download:
https://github.com/Wacom-Developer/stu-sdk-samples

STEP-BY-STEP INSTRUCTIONS
--------------------------

1. Visit the Wacom Developer documentation site:
   https://developer-docs.wacom.com/docs/stu-sdk/windows-sdk/

2. Download the Windows SDK (x64 version)
   - Look for "Wacom STU SDK for Windows (x64)"
   - Recommended version: v1.4 or later

3. Extract the downloaded ZIP file

4. Navigate to the SDK binaries folder:
   stu-sdk-windows\sdk\bin\win64\

5. Copy the following files to this directory (C:\Users\juanr\FirmasApp\libs\):
   - wgssSTU.dll
   - wgssSTU_x64.dll (if available)

6. Build and run the application

VERIFICATION
------------
After copying the DLLs, you can verify the installation by:

1. Building the project: dotnet build
2. Running the application: dotnet run
3. Check the debug.log file for connection status

TROUBLESHOOTING
---------------
- If you see "DLL not found" errors, verify the DLLs are in this directory
- If you see "DLL loading error", ensure you have the 64-bit versions (not 32-bit)
- If no device is detected, check that:
  * The Wacom STU-430 tablet is connected via USB
  * Device drivers are installed
  * The tablet is recognized in Windows Device Manager

FALLBACK MODE
-------------
If the DLLs are not present or the device is not connected, the application
will automatically fall back to simulation mode, allowing you to test the
signature capture UI without a physical device.

CONFIGURATION
-------------
The application can be configured to use simulation mode even when the DLLs
are present by modifying appsettings.json:

{
  "WacomStu": {
    "UseSimulation": true
  }
}

SUPPORT
-------
For issues with the Wacom STU SDK itself, refer to:
https://github.com/Wacom-Developer/stu-sdk-samples/issues

For issues with this application's integration, check the debug.log file
in: %LocalAppData%\FirmasApp\debug.log
